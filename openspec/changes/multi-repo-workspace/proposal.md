## Why

Developer tasks routinely span multiple repositories — an API change requires updating the frontend, shared proto definitions, and sometimes infrastructure config. The current system supports multiple repos per AgentRun at the infrastructure level (hydrator, workflow), but the agent experience is single-repo-centric: it starts anchored in the first repo's directory, devbox runs only against the primary repo's config, and the UI only lets you pick one repo. We need to make multi-repo the default mental model, not a special case, and let the agent set up its own development environment using devbox across the full workspace.

## What Changes

### Backend: Workspace Environment
- **Agent starts at `/workspace` root** instead of being dropped into the first repo's directory, with a generated `uncspace.yaml` manifest describing the workspace layout
- **Hydrator auto-composes devbox environments** by scanning each cloned repo for `devbox.json` and generating a root-level `devbox.json` that includes them all via `devbox include`
- **Agent can self-serve devbox** — install additional packages it discovers it needs via `devbox add`, filling gaps that individual repo configs don't cover
- **`uncspace.yaml` workspace manifest** generated by hydrator at `/workspace/uncspace.yaml`, describing repos, paths, branches, and devbox source configs — the agent's map of its workspace
- **Sidecar runs agent from `/workspace`** with the composed devbox environment instead of a single repo's devbox shell
- **Workflow passes `/workspace` as working directory** instead of computing primary repo path

### Proto/CRD: Workspace Name
- **Add `workspace_name` field to `AgentRunSpec`** — optional string that tags a run with the workspace preset it was created from, enabling UI filtering by workspace

### UI: Multi-Repo & Workspace Presets
- **Fix type flattening** — web types carry full `repos[]` array instead of flattening to single `repoURL`
- **Workspace presets (localStorage)** — saved collections of repos + branches that users can select when creating runs, avoiding repeated multi-repo configuration
- **Workspace editor** — create, edit, delete workspace presets with name, description, and repo list
- **Sidebar workspace section** — filter runs by workspace, alongside existing phase and repo filters
- **Redesigned AgentRunForm** — workspace selector, multi-repo with per-repo branches, add/remove repos
- **Multi-repo display** — table and detail panel show all repos per run, not just the first
- **Repo registry** — ReposView becomes interactive with add/remove, not just read-only

## Capabilities

### New Capabilities
- `workspace-manifest`: Generation and structure of the `uncspace.yaml` file that describes the agent's workspace — repos, paths, branches, devbox sources
- `devbox-composition`: Auto-composition of per-repo devbox.json configs into a unified root-level environment, conflict detection, and agent self-serve package installation
- `workspace-presets`: Client-side workspace presets for saving and reusing multi-repo configurations, including the workspace editor, sidebar filtering, and form integration
- `multi-repo-ui`: Web UI changes to support multi-repo display, selection, and filtering across all components — types, form, table, detail panel, sidebar, search

## Impact

- **Hydrator** (`internal/hydration/hydrator.go`): New step to scan for devbox configs, generate `uncspace.yaml`, compose root `devbox.json`, run `devbox install` at workspace root
- **Workflow** (`internal/temporal/workflow.go`): Change `workspacePath` computation from first-repo-based to `/workspace`
- **Sidecar** (`internal/sidecar/gateway.go`): `startAgentProcess` uses `/workspace` as working dir, runs `devbox run` from root
- **Proto/CRD** (`proto/aot/api/v1/api.proto`, `api/v1alpha1/types.go`): Add `workspace_name` field to `AgentRunSpec`
- **Web UI types** (`web/src/types/agent-run.ts`): Replace `repoURL: string` with `repos: Repository[]`, add `workspaceName`
- **Web UI hooks** (`web/src/hooks/useClient.ts`): Stop flattening repos in `mapRun()`, pass through full array
- **AgentRunForm** (`web/src/components/AgentRunForm.tsx`): Full redesign — workspace selector, multi-repo with per-repo branches
- **AgentRunTable** (`web/src/components/AgentRunTable.tsx`): Multi-repo column display with truncation
- **AgentRunDetailPanel** (`web/src/components/AgentRunDetailPanel.tsx`): Repo list instead of single repo
- **Sidebar** (`web/src/components/Sidebar.tsx`): New Workspaces section, multi-repo-aware repo counts
- **ReposView** (`web/src/components/ReposView.tsx`): Interactive repo registry with add/remove
- **App.tsx**: Multi-repo filtering, workspace state, search across all repos
- **New components**: WorkspaceEditor modal, localStorage persistence hook
